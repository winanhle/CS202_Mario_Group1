### Update Cycle (Per Frame)
```

?   ??? Query: getPlayerPosition() ? Player position
?

?   ??? Query: getPlayerPosition() ? Player position

?   ?
?   ??? Query: getEnemyCount() ? How many enemies?
?   ?
?   ??? Query: getItemCount() ? How many items?
?   ?
?   ??? Notify: hudManager?updateScore/Lives()

```
GameWorld.render(window)

## Module Communication Patterns

### Pattern A: Module Queries Another Module
```
Source: PlayerManager::update()
?
??? m_gameWorld->getEnemyManager()  [Get reference to interface]
    ?
    ??? IEnemyManager* enemies = m_gameWorld->getEnemyManager()
        ?
        ??? if (enemies->getEnemyCount() > 0) { /* handle */ }
```

### Pattern B: Module Notifies HUD
```
Source: PlayerManager::update() [When score changes]
?
??? m_gameWorld->getHUDManager()  [Get reference to interface]
    ?
    ??? IHUDManager* hud = m_gameWorld->getHUDManager()
        ?
        ??? if (hud) { hud->updateScore(m_score); }
```

### Pattern C: GameWorld Coordinates Updates
```
Source: GameWorld::update()
?
??? For each manager that needs coordination:
?   ??? IPlayerManager* player = m_playerManager.get()
?   ??? IHUDManager* hud = m_hudManager.get()
?   ?
?   ??? if (player && hud) {
?       hud->updateScore(player->getScore());
?       hud->updateLives(player->getLives());
?   }
```

## Thread Safety Notes

Current design:
- **Single-threaded** - All operations on main thread
- **No locks needed** - Game loop controls all access
- **Update order guaranteed** - GameWorld controls sequence

If multi-threading is added later:
- Each manager would need its own mutex
- GameWorld would synchronize access
- Interfaces would remain unchanged

## Performance Characteristics

### Per-Frame Operations
```
GameWorld::update(dt)
?? EnemyManager::update()     O(n) where n = enemy count
?? ItemManager::update()      O(m) where m = item count
?? PlayerManager::update()    O(1) + collision checks
?? HUDManager::update()       O(1) for text updates

GameWorld::render()
?? EnemyManager::render()     O(n)
?? ItemManager::render()      O(m)
?? PlayerManager::render()    O(1)
?? HUDManager::render()       O(1) text rendering
```

### Memory Usage
```
Per manager instance:
- Interfaces: ~0 bytes (virtual table)
- Managers: Depends on implementation
- Typical: <1MB per manager for reasonable entity counts
```
